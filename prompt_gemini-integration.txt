Integration phase with gemini:

Job: You are a senior software architect in Node.js and integration with gemini-2.5-flash.

Task:
Create a Node.js (Express) server side, using the official Gemini SDK (@google/genai).

Background:
Currently, the application (client side) has a mechanism that presents static and fixed diagnostic questions to the user. I want to upgrade the system and replace this static logic, so that the server side will contact the Gemini API and generate questions and diagnostics in a dynamic and intelligent way.

Technical requirements:

Fallback mechanism: An asynchronous function that will try to contact the gemini-2.5-flash-lite model.

If a load error (429/Quota) is received, the system will automatically switch to the gemini-2.5-flash model.

Built-in output: ResponseSchema must be used to receive clean JSON only.

Application paths (POST):

/api/generate-questions:
Path without body. The server will ask Gemini to generate 4 general and varied American questions in Hebrew for an initial occupational diagnosis (4 options for each question). It will return a JSON array of the questions.

/api/analyze-results:
Receives in body an array of the 4 questions and the answers that the user selected. The server will ask Gemini to analyze the choices
and return in JSON the 3 most suitable professions for him (profession name, percentage of match and explanation).

Code guidelines:
Use ES Modules, support CORS, and load the API key from a .env file (named GEMINI_API_KEY).

Write the program according to standard node.js practices

Create a PLAN for this program.

Task:
Add a preliminary step of free text input from the user before creating the American questions. The free text will serve as the basis for creating customized questions by Gemini.

Background
Currently, the system starts directly by creating fixed American questions without personal input from the user. It is necessary to change the flow so that the questions are created dynamically according to the description that the user enters about himself.

Technical characterization
The existing endpoint /api/generate-questions will be updated to receive a userText field in the request body.
The server will send the userText to Gemini to create 4 customized American questions, with each question having 4 answer options.
There is no change in the endpoint for analyzing the results /api/analyze-results.
After the user answers the questions, the answers along with the userText will be sent to the server for analysis in Gemini and returning 3 suitable professions with matching percentages and an explanation.

Continue to use @google/genai, support fallback between models, use responseSchema, ES Modules, CORS, and load an API key from an .env file called GEMINI_API_KEY.
