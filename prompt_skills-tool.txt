Step using the TOOL Calculating given talents that optimizes the system results!

Role:

 You are a Senior Software Architect specializing in Node.js, Agentic AI and Google Gemini.

Goal:

Expand the recommendation mechanism so that the final recommendation is not based only on the conversation history, but also on the analysis of the skills identified in the user using a dedicated Tool.

Background:

Currently, the Agent has a conversation with the user, asks questions and finally returns recommendations.

At this stage, after the Agent decides that enough information has been collected, it must run a Tool that analyzes the entire conversation history and returns a list of skills with a percentage of matching for each skill.

After receiving the results of the Tool, the Agent will receive the skills along with the conversation history, and will use both sources of information to return one final and more accurate recommendation.

Technical requirements:

An Agent Loop must be implemented in which the model decides whether to ask another question, run a Tool or return an answer Final.

Loop Control must be implemented to prevent infinite loops.

Create a Skills Analysis Tool that receives the conversation history and returns a list of skills with a matching percentage.

The server will execute the Tool when the Agent requests it, return the result to the Agent, and only then will the Agent return a final recommendation for one profession.

The client side will continue to present questions as long as additional information is required, or the final recommendation when the process ends.

Expected result:

A more accurate and smarter recommendation system, in which the final recommendation is based on both the content of the conversation and the analysis of the skills calculated using the Tool.

Create a complete and professional PLAN before writing the code, and follow Best Practices and modular architecture.
